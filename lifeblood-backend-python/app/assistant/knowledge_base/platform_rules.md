# LifeBlood Platform Usage Rules

## Account and roles
- Users register with name, email, password, phone, blood group, and address (division, district, upazila).
- Each account has a role: donor, recipient, or admin.
- New accounts must be verified before some actions are fully enabled; admins can verify, disable, or reactivate user accounts.

## Searching for donors
- Any visitor can use the "Find Donors" search to look up donors by blood group, division, district, and upazila.

## Blood requests
- Logged-in users can create a blood request specifying blood group needed, location, and a deadline.
- The request notifies nearby compatible donors automatically.
- Donors who receive a request notification can accept or decline it from their Notifications page.
- Requesters can track the status of their requests (open, fulfilled, cancelled) under "My Requests".

## Chat between users
- Users can chat directly with another user (for example, after a donation request connection) from the Chat section.
- Users can turn their own chat availability on or off from Chat Settings; if a user has chat turned off, others cannot message them.
- Users can block another user; once blocked, that user cannot send messages either way.
- Chat supports text messages and sharing a location.

## Donation history and badges
- Donors can log their past donations (date, location, optional recipient contact and notes) from their Dashboard.
- The system tracks total donation count and last donation date, and awards badges ("Hero Donor", "Life Saver") based on donation count.

## Respecting other users
- Do not use the chat or contact information obtained through the platform to harass, pressure, or spam other users.
- Always verify medical eligibility with a doctor when in doubt rather than relying solely on the platform's general guidance.
